How it works

This uses the Web Serial API — your browser talks directly to your Arduino over USB. No drivers, no extensions, nothing to install. Just plug in, click Connect, and pick the port.

1
Plug in your board
USB cable from your Arduino to your computer. If the IDE sees it, ArduinoLab will too.
2
Close the Arduino IDE
Only one program can hold a serial port at a time. If the IDE's Serial Monitor is open, close it first.
3
Click Connect
Your browser will ask which port to use. Pick your Arduino (often labeled COM8, /dev/ttyACM0, or similar).

Want to test it right away? Upload this sketch — it sends a fake temperature and light reading every 100ms. The plotter will pick up both values automatically.

// ArduinoLab demo — prints two values every 100ms
// Open Live Monitor, click Connect, pick this board

void setup() {
  Serial.begin(9600);
}

void loop() {
  int temp = analogRead(A0);          // pretend temp sensor
  int light = analogRead(A1);         // pretend LDR
  Serial.print("temp:");
  Serial.print(temp);
  Serial.print(",light:");
  Serial.println(light);
  delay(100);
}

The plotter auto-detects numbers. It handles plain numbers (523), CSV (12,45,67), and labeled pairs (temp:23,light:812) — up to six series at once.

my LED isn't lighting up even though I copied the diagram

    
Happy to help. Let's check the usual suspects:

1. Polarity. LEDs only work one way. The longer leg is positive — make sure it's on the pin 13 side, not GND.
2. Resistor. Are you using a 220Ω? Too high and the LED stays dim or dark.
3. Pin. Double-check the wire is in D13, not A13 or VIN.

Which one do you want to check first?
